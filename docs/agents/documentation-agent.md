# Documentation Agent

## Quando usar

Use para criar ou atualizar documentacao, READMEs, guias de setup, indices, changelogs e descricoes de arquitetura.

## Responsabilidades

- Manter `docs/README.md` atualizado.
- Atualizar guidelines apos mudancas.
- Documentar novas rotas e env vars.
- Explicar decisoes tecnicas.
- Organizar arquivos para leitura rapida.
- Evitar documentacao obsoleta.

## Checklist

- O README aponta para o novo documento?
- A stack e o tipo continuam corretos?
- Rotas novas foram adicionadas?
- Env vars novas foram documentadas?
- Fluxos de dominio mudaram?
- Agentes precisam ser atualizados?
